# Риски

## Продуктовые риски

### Продукт не нужен пользователю

Люди привыкли сами строить маршруты или использовать уже знакомые сервисы.

**Минимизация:** более глубокое исследование перед началом работы — интервью с пользователями, детальный анализ конкурентов, анализ поисковых запросов, тестирование прототипа.

### Низкая возвращаемость

Пользователь использует продукт 1–2 раза в год.

**Минимизация:** фокус на жителях города, а не только на туристах. Тематические маршруты (вечерние, на один день, для свиданий). Push-уведомления с новыми маршрутами и регулярные публикации в соцсетях (подборки, истории).

## Технологические риски

### Проблемы с геолокацией

Особенно актуально для РФ — во многих городах плохо работает геолокация.

**Минимизация:** офлайн-карты, текстовый режим маршрута, пошаговый список с точными ориентирами.

### Блокировки интернета

Возможна как полная, так и частичная блокировка интернета.

**Минимизация:** загрузка маршрута офлайн, кэширование карт.

## Юридические риски

### Персональные данные

Геолокация — это персональные данные (152-ФЗ «О персональных данных»). Вопрос требует тщательного изучения.

**Минимизация:** подготовить политику конфиденциальности, не хранить историю перемещений, хранить геолокацию только в момент сессии.

### Авторские права на контент

Есть риск использовать в описании маршрутов или в рекламе фотографии, описания мест и исторические материалы, на которые действуют авторские права.

**Минимизация:** использовать собственный контент и лицензионные изображения, указывать источники информации.

## Рыночные риски

### Сильная конкуренция

2GIS, Яндекс Карты — это гиганты. Spot не должен стать ещё одной картой, а должен чётко выделиться.

**Минимизация:** чётко выделить УТП и использовать его в рекламе и маркетинговых кампаниях. На первых порах больше сотрудничать, чем конкурировать.

### Сложность монетизации

Реклама внутри приложения может портить UX, а подписки тяжело продавать без уникальных и нужных пользователю функций.

**Минимизация:** на этапе discovery продумать функции, которые стоит оставить для подписки. Сделать рекламу нативной и интегрировать её в маршруты.

## Операционные риски

### Масштабирование контента

Чтобы расти, нужно больше городов и маршрутов, а также их регулярное обновление — это затратно по трудозатратам и времени.

**Минимизация:** редакторы в разных городах, которые делают новые качественные маршруты. Мотивировать пользователей добавлять свои маршруты через систему наград за достижения.
