## Акторы и характер взаимодействия

| Актор | Действия | Задействованные сущности | Характер взаимодействия |
| --- | --- | --- | --- |
| **Пользователь** | Регистрация, авторизация, просмотр каталога, навигация, оценки, избранное | Пользователь, Сессия, Оценка, Избранное | Читает: маршруты, точки. Пишет: регистрацию, оценки, избранное |
| **Пользователь (как автор)** | Создание и публикация маршрутов и точек, управление профилем | Автор, Маршрут, Точка маршрута | Пишет: маршруты, точки. Читает: свои маршруты, статистику |
| **Мобильное приложение** | Запросы к каталогу, геолокация, кэш, формирование ленты | Маршрут, Точка, Пользователь | Читает массово: каталог, «Популярное», «Поблизости» |
| **Аналитик** | Анализ вовлечённости, популярных маршрутов, городов, конверсии | Маршрут, Оценка, Пользователь (агрегаты) | Только чтение, сложные OLAP-запросы |
| **Администратор** | Модерация контента, управление пользователями, мониторинг | Все сущности | Чтение и запись во все сущности |

## Транзакционные и аналитические данные

### Транзакционные данные

- Регистрация и авторизация пользователя — атомарность обязательна.
- Публикация маршрута с точками — единая транзакция (маршрут + N точек).
- Добавление оценки / избранного — запись с FK-контролем уникальности.

### Аналитические данные

- Топ популярных маршрутов по городу, рейтинг по категориям.
- Конверсионные воронки: регистрация → просмотр → навигация → оценка.
- Поведение пользователей: среднее число сессий, повторные визиты, отток.

## Выбор технологий хранения

| Шаг | Пользователь | Маршрут | Точка маршрута | Оценка / Избранное | Сессия |
| --- | --- | --- | --- | --- | --- |
| **Объём данных** | 50 000–500 000 строк (~50 МБ) | 250 000 маршрутов (~500 МБ) | ~2,5 млн точек (~2 ГБ) | ~5 млн записей (~1 ГБ) | ~10 млн активных сессий (~2 ГБ) |
| **Обоснование оценки** | 50 000 регистраций/мес × 3 года | 250 000 маршрутов; рост по мере добавления авторов | ~10 точек на маршрут; рост линейно с маршрутами | Предположение: 20 действий на пользователя | 200 000 одновременных сессий; TTL 30 дней |
| **Паттерн доступа** | OLTP: регистрация, авторизация, чтение профиля по ID | OLTP (чтение каталога) + OLAP (топ по городу, рейтинг) | OLTP: чтение точек по ID маршрута | OLTP write-heavy: запись оценки, добавление в избранное | Key-Value: get/set по токену |
| **Консистентность** | Strong (ACID): данные аккаунта, хеш пароля | Strong для публикации; Eventual для рейтинга (агрегат) | Strong: структура маршрута должна быть консистентна | Eventual: рейтинг обновляется агрегатом, не критично | Weak: сессия — кэш |
| **Доступность** | 99,9%; одной реплики достаточно | 99,9%; каталог — ключевая функция | 99,9% вместе с маршрутом | 99,5%; рейтинг можно скрыть | 99,9%: без сессии нет доступа к продукту |
| **Мутабельность схемы** | Жёсткая: поля аккаунта редко меняются | Полужёсткая: маршрут может расширяться (новые теги, медиа) | Жёсткая: координаты, стабильная структура | Жёсткая: оценка 1–5 | Жёсткая: токен + TTL + device |
| **Транзакции** | Да, критично: создание аккаунта (атомарность) | Да: публикация маршрута + точек — одна транзакция | Да: вставка точек маршрута вместе с маршрутом | Частично: запись оценки атомарна, но обновление рейтинга — нет | Нет: set/get, TTL транзакции не нужны |
| **Запросы и поиск** | Простой: поиск по номеру телефона, по ID | Сложный: фильтр по городу, тегам, рейтингу; сортировка | Простой: все точки маршрута по ID | Простой: есть ли оценка | Простой: get по токену |
| **Итоговое решение** | **PostgreSQL** | **PostgreSQL** | **PostgreSQL** | **PostgreSQL** | **Redis** (in-memory key-value, TTL, быстрый доступ по токену) |
