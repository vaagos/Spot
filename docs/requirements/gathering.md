# Сбор требований

## RACI-матрица стейкхолдеров

**Функционал:** добавление маршрута пользователем и публикация после модерации.

| Стейкхолдер | R | A | C | I |
| --- | --- | --- | --- | --- |
| Пользователь | R (создание маршрута) | | | I |
| Модератор | R (проверка контента) | | | I |
| Product Manager | | A | C | |
| Backend-разработчик | R (реализация логики) | | | |
| Mobile-разработчик | R (UI создания маршрута) | | | |
| UX/UI дизайнер | R (UX/UI интерфейса) | | | I |
| Юрист | | | C | I |
| Маркетинг | | | | I |
| Руководитель проекта | | A (финальное бизнес-решение) | | I |

### Расшифровка ролей

- **R = Responsible** — тот, кто непосредственно выполняет задание.
- **A = Accountable** — тот, кто принимает работу и несёт ответственность за результат.
- **C = Consulted** — тот, кто оказывает консультативную помощь.
- **I = Informed** — тот, кто в курсе принимаемых решений и хода выполнения задачи.

### Пользовательский опыт

| № | Вопрос | Цель |
| --- | --- | --- |
| 1 | Как вы представляете идеальный опыт прохождения маршрута в приложении? | Выявить ожидания по UX |

| № | Вопрос | Цель |
